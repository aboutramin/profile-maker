====================
FIELD GUIDELINES
====================

candidate_identity:
Use only directly supported identity information.
If a field is not supported by evidence, leave it empty.
Do not guess target roles unless the source documents clearly support them.

languages:
Include only languages that appear in the source evidence.
Use compact entries, for example:
[
  "English",
  "Finnish",
  "Persian"
]
Do not guess proficiency levels unless they are explicitly supported.

certifications:
Include only verified certifications, courses, licenses, or formal training that are career-relevant.
Do not list normal degrees here; use education for degrees.

education:
Keep concise.
Include degree, field, institution, location, dates, and thesis/topic only when useful and supported.

career_timeline:
Keep concise.
Include roles, organizations, locations, dates, and compact responsibility keywords.
Do not write narrative job descriptions.

verified_projects:
* maximum 20 projects
* use keywords only
* no long descriptions
* include tools and domains only when supported by evidence
* merge similar projects when possible

verified_skills:
* deduplicated
* maximum 100 entries
* use compact skill names
* do not include weak or vague claims unless evidence is clear

verified_software:
* deduplicated
* maximum 50 entries
* include software only when supported by evidence
* do not infer software from general domain knowledge

verified_methods:

verified_domains:

verified_achievements:

Avoid exaggeration.
Avoid vague praise.

verified_strengths:
Use only repeated or clearly supported professional strengths.

uncertain_claims:
