OBJECTIVE

Improve quality while preserving truthfulness.

Keep only information useful for:

- CV tailoring
- Cover letter generation
- Truthfulness auditing

====================
RULES
====================

1. Keep all supported high-value information.
2. Add missing verified information from the review only when it is supported by evidence.

9. Do not invent facts.
10. Do not expand narrative content.
11. Preserve the same JSON structure as PROFILE_V1.
12. Do not add new top-level fields unless they already exist in PROFILE_V1.
13. Do not remove required top-level fields.
14. Do not rename top-level fields.
15. Do not add commentary outside JSON.
16. Compression is important.
17. If the review recommends an addition but the evidence is unclear, put it in uncertain_claims or leave it out.
18. If the review conflicts with PROFILE_V1 and no source evidence is provided, choose the safer and more conservative option.
19. Keep evidence_index compact.
20. Use DOC IDs in evidence_index when source references are available; do not depend on original file names.
